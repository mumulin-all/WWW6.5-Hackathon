**第一页：点击某个信号弹后进入的信号链**

**时空长廊 (The Chain Corridor)或者说是信号链，代表信号的上下游**

点击某个信号后进入，展示该信号的“家族血脉”。

- **视觉效果：** 纵向排布。中间是焦点信号，上方是**母信号**（淡出），下方是**衍生信号**（缩小）。一根琥珀色细线贯穿全场。

- **功能逻辑：**

- - **向上滑动：** 追溯母信号（孕育了当前信号的源头）。
  - **向下滑动：** 查看衍生信号（受当前信号启发而诞生的新信号）。
  - **视角切换：** 点击任何一个母/子节点，它会平滑移动到屏幕中心成为新的主角。

- **完整信息：**

- - 每个节点带作者地址（0x...）、发射时间、区块高度。

**第三页：回答作者的问题**

**这是你最看重的“安全感”与“内心暴露”的发生地。**

- **设计理念：** “对坐交谈”。像是在深夜的森林里，对着一团篝火，把秘密托付给对面那个模糊的身影。

- **视觉布局：**

- - **顶部：** 半透明悬浮母信号的“影子”与“问题”，作为思考的引航灯。
  - **中央：** 无边框的纯净文本域，琥珀色字体，打字时带有细微的光晕。
  - **边缘：** 极其微弱的身份水印（你的钱包地址），提醒你这一刻的真实。

- **核心功能：**

- - **输入：** 针对引子的共鸣或对问题的回答。
  - **动作：** 按钮命名为 **“回答”**。

- **后端逻辑：**

- - 点击“回答”后，内容上传      IPFS，触发钱包签名。
  - 该记录**不产生新信号链，仅是读者回答作者的问题，让作者选择是否放开文章权限。**

**第四页：回答发出**

发射成功后的瞬间反馈。

- **展示：**

- - Status: Successfully Inscribed (成功铭刻)。
  - 展示本条回响的 **IPFS CID**。
  - 展示 **Transaction Hash**。

- **情感反馈：** 给读者一种“我的软弱已被星空接纳并永恒保存”的安全感。

**第五页：进入阅读文章页面**

**引爆与沉浸阅读 (The Ignition)**

- **入口仪式 (The Invitation)：**

- - 界面中心漂浮一个琥珀色发光球体（Floating Sphere），具备呼吸灯效果和轻微的上下位移物理感。
  - **交互：** 点击球体触发“烟花炸开”动效（粒子向四周放射并消失），随后文章正文以 2000ms 的时长平滑淡入。

- **阅读体验与沙漏 (The Sands of Time)：**

- - **取消倒计时：** 顶部仅保留一个缓慢旋转的**沙漏图标**，作为时间的视觉隐喻，严禁使用数字倒计时，以消除读者的心理压力。
  - **挽留机制 (Retention Logic)：** 文章**不会**因时间到而强制关闭。系统需监测读者的行为：只要读者停留在阅读区域、正在滚动页面、或鼠标处于活跃状态，文章将始终保持可见。

**第六页：阅后感流向选择**

**回响与选择流向 (The Shaper of Echoes)**

- **触发条件：** 读者阅读完正文后，页面自动下滑或淡入“阅后感”区域。

- **引导文案：** “想到什么就写下来吧，不用完整，不用漂亮，她也想收到你的信号。”

- **流向选择 (Three Paths)：**

- - **“****悄悄送给作者”：**

  - - **视觉：** 文字收缩聚合成一只 **纸鹤 (Paper Crane)**，向屏幕右上方飞出并淡出。
    - **逻辑：** 写入私密数据库，仅作者可见。

  - **“****留在文章下”：**

  - - **视觉：** 文字聚合成一颗 **星星       (Shimmering Star)**，在原地闪烁后融入背景。
    - **逻辑：** 转换为公开评论/回响，附着于该文章哈希下。

  - **“****发射出我的信号弹”：**

  - - **视觉：** 页面平滑转场，直接带入“创建新信号（引子与问题）”界面。
    - **逻辑：** 建立信号溯源关系，当前文章       ID 自动关联为新信号的“母信号”。

**3.** **退出逻辑 (The Departure)**

**四、 给开发队友的实现建议 (For Developers)**

 

**💡** **总结给队友的开发要点：**

**给队友的单独说明文档**

**项目名称：** Semaphore (信号协议) 原型说明 

**1.** **读者浏览信号链和回答问题**

**交互设计规范** **(Interaction Spec)**

•  **初始进入：** App.tsx 中的 hasMoved 标志位控制主信号的首次强制点亮。只要检测到一次 onMouseMove，系统就进入“动态感应态”。

•  **视线追踪** **(Proximity Detection)****：** 在 CorridorView.tsx 中，通过计算 mouseY 与各卡片中心点的垂直偏移量，动态切换卡片透明度。**注意：** 当鼠标直接 hover 在卡片上时，CSS 的 group-hover 会覆盖 JS 的计算结果，确保交互不冲突。

•  **连接点外置：** .connection-point 元素使用 fixed 居中的琥珀线作为坐标参考。它是卡片的一个兄弟节点，但视觉上锚定在线条上。

**2****，阅读文章和阅后感流向选择**

1. **状态机驱动：** 建议在 EchoChamberView 组件中使用 Stage 状态管理：floating (球) -> exploding (烟花) -> reading (阅读) -> shaping (写作) -> sealing (转场)。

2. **挽留逻辑实现：**

3. - 使用 useEffect 配合 mousemove 和 scroll 监听器。
   - 在读者点击“封缄”按钮前，锁定      onBack 回调，确保文章陪伴读者完成输入。

4. **动画性能：**

5. - “烟花”粒子建议使用 CSS 动画实现（transform:      translate(var(--tx), var(--ty))），以保持低能耗。
   - “纸鹤”与“星星”的转场仪式建议持续      2000ms - 2500ms，给用户足够的情感留白时间。
