# 📜 Alcheme (炼金与自我) 产品需求文档 (PRD)

---

**项目名称：** Alcheme

**项目愿景：** Refine fragments, awaken true self.

**核心哲学：** *What you do and how you think build who you are. Define yourself, beyond the rules.* 我们不预设规则，你才是自己人生的炼金术士。

---

## 一、 项目背景与深度痛点 (The "Why")

### 1.1 Web2 时代的“数据流放”

- **数据孤岛与资产流失**：我们在各平台产生的阅读、运动、代码数据被困在巨头应用里。这些代表我们“成长资产”的数据，用户却不拥有其主权。
- **自我认知的碎片化**：现代人很难看到一个“多维、真实、成长中”的自己。那些微小的进步（比如坚持学习 3 天编程）转瞬即逝，缺乏正向反馈，导致成就感虚无与内耗。

### 1.2 社会标签的“垂直压迫”

- **过度标签化**：传统的简历和社交媒体强迫我们变得“垂直”且“标签化”（如：XX经理、XX程序员）。
- **多维生命体的抹杀**：社会规则要求我们职业化，但人是复杂的。斜杠青年、跨界爱好者需要一个能打破既定框架、自定义身份的工具。**你是什么样的人，只有你自己说了算。**

### 1.3 解决方案：炼金术逻辑

利用 AI 提取日常碎片的灵光，通过区块链技术将其凝练为不可篡改的 **SBT (Soulbound Token)** 勋章。它不是一份简历，而是一面反映真实自我、可进化的“数字镜子”。

---

## 二、 核心产品逻辑 (The Alchemy Pipeline)

我们构建了一个**“晶石 (Ore) → 卡牌 (Card) → 勋章 (Badge)”**的进化链条，强调**“个人定义”**与**“量化增长”**。

1. **采集 (Collecting - 晶石)**：
    - 记录任何一件小事、观点或微小进步。
    - **定义权**：无论多小的事，只要你认可，它就是一颗闪光的“晶石”。
2. **精炼 (Refining - 卡牌)**：
    - 当若干晶石汇聚成一个阶段性成果（哪怕只有一颗极重要的晶石），即可熔炼成“里程碑卡牌”。
    - **视觉激励**：AI 根据内容生成绝美的艺术图片。
3. **觉醒 (Awakening - 勋章)**：
    - 多个里程碑卡牌合成一个“身份勋章”（如：Web3 探险家）。
    - **上链存证**：勋章作为 SBT 铸造，包含所有底层的晶石与卡牌信息。
4. **进化 (Evolution - 持续增长)**：
    - 身份不是静态的。你可以不断向已有勋章注入新卡牌，让勋章随你一同进化。

## 三、 功能模块详述 (The 4-Page Demo)

### Page 1: 采集 (Collect) - 灵感矿山

- **交互一：矿山挖掘**。用户输入语音/文字，AI 实时归纳提炼，生成一颗颗刻有核心信息的**晶石 (Ore)**。
- **交互二：外部信纸**。模拟外部应用传入的数据，点击“信纸”即可将其作为晶石直接收集，打破数据孤岛。

### Page 2: 精炼 (Refine) - 炼金锅炉

- **视觉**：中央是一个燃烧的锅炉，侧边是存放晶石的橱柜。
- **动作**：从橱柜选中想要熔炼的晶石（可多选，也可单选），点击 Refine。
- **产出**：AI 自动生成总结性文字与精美的**卡牌插图**。用户可微调内容，给予自主定义权。

### Page 3: 觉醒 (Awaken) - 神秘之门

- **视觉**：一扇通往自我深处的神秘大门，背后陈列着已有的卡牌与勋章。
- **动作**：勾选多张卡牌“觉醒”为全新勋章；或选中已有勋章并叠加新卡牌进行 **Evolve（进化）**。
- **核心逻辑**：勋章是身份的证明，也是成长轨迹的浓缩。

### Page 4: 档案 (Profile) - 勋章墙与六维镜

- **勋章墙**：对外展示的核心。点击每一枚勋章，均可追溯其背后的卡牌与晶石，形成**不可作假、全透明的成长 Proof**。
- **六维雷达图**：
    - **逻辑**：用户自定义维度，AI根据晶石内容AI 自动评分。它不用于与他人比较，只用于观测自己的多维增长。
    - **理念**：拒绝单一维度的内卷，鼓励成为自定义的“六边形战士”。

## 四、 技术规格 (Tech Specification - Demo Version)

- **前端交互**：
    - **开发范式**：**AI-Driven Component Synthesis (AI 驱动的组件合成)**。我们弃用了传统的“手动像素级编码”，采用 **Figma + Gemini 3** 的混合工作流，通过自然语言指令直接驱动 UI 逻辑的生成。
    - **核心架构**：**React / Next.js**。
    - **样式处理**：**Tailwind CSS**。
    - **UI 风格**：**“轻量炼金女巫风”**。以紫色、淡紫色、白色、金色为主色调，清新且具有神秘感，辅以可爱的卡通炼金元素。
- **AI 接口**：采用双大模型策略
    - **LLM**：OpenAI GPT-4o (处理语义提炼与评分)。
    - **Image Gen**：gpt-image-1（DALL-E 图像生成） (生成卡片艺术图)。
- **区块链 (Smart Contract)**：
    - 采用 **ERC-721** 协议（SBT 模式）。
    - **存储逻辑**：晶石与卡牌暂存线下（Local/Off-chain），勋章铸造时将元数据打包上链。
    - **Proof 属性**：支持社交展示与招聘/合作伙伴的真实性验证。

## 五、 产品独特哲学 (The Alcheme Difference)

1. **非规则化 (Unstructured)**：我们不设等级，不设标准模板。你的成长由你定义数量、定义时机。
2. **抗对比性 (Anti-Comparison)**：雷达图与勋章墙仅针对个人进化的视觉反馈，强调“多维自我”的丰富度，而非社会层级的优劣。
3. **可追溯的真实 (Traceable Truth)**：利用 Web3 特性，让每一份荣耀（勋章）背后都有扎实的汗水（晶石）支撑。

### 💡 针对 Demo 阶段的思考（局限性与补齐）

- **关于规则滥用的思考**：虽然我们鼓励自由定义，但在将来实际应用中，可以通过**“社区共识”**或**“AI 逻辑一致性校验”**来防止恶意灌水（例如：不能用一句话重复生成 100 颗晶石）。
- **SBT 延迟问题**：在 Demo 演示时，我们将通过前端的**“即时视觉反馈”**（如勋章瞬间变亮、变大）来弥补链上数据刷新的延迟感。
- **数据接入**：目前的 Mock 模拟是为了展示将来“全数据接入”的可能性，体现我们打破 Web2 孤岛的决心。
